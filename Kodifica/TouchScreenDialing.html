Kodifica 1000 TouchScreenDialing

Problem Statement

Your company is marketing a phone that allows input via a touch screen. The pressure sensitive screen sends coordinates to the processing device, which then determines which numbers have been input. In this problem, you are writing the code for the processing device. You will be given int[]s rows and cols specifying where the screen has been pressed. More precisely, the ith press occurred at row rows[i] and column cols[i] (both 0-indexed). You are also given a String[] screen containing the image the user sees as they dial. Each element of screen is a row of the image, and each character of a single row is in a different column. screen will be divided up into numerous regions. Two points of the screen are in the same region if you can get from one to the other using only horizontal or vertical steps that never cross an X character. For each of the numerals '0'-'9', there will be a region containing that numeral. A press that occurs anywhere in a region containing a numeral results in that number being dialed. A press that occurs in a region containing no numeral, or directly on an X character results in no dialing. Other than numerals and Xs, screen will also contain (quotes for clarity) '.' characters as fillers. Your method will return a String containing the numbers dialed by the user (see example 1 for further clarification).

(be sure your method is public)

Limits

  • Time limit (s) 2.000
  • Memory limit (MB) 64

Constraints

  • screen will contain between 1 and 50 elements inclusive.
  • Each element of screen will contain between 1 and 50 characters inclusive.
  • Each element of screen will contain the same number of characters.
  • The characters in screen will all come from the following string (quotes for clarity): "0123456789.X".
  • Each digit (0-9) will occur exactly once in screen.
  • Each digit will be the only digit in its region.
  • rows will contain between 1 and 50 elements inclusive.
  • cols will contain the same number of elements as rows.
  • Each element of rows will be between 0 and N-1 inclusive, where N is the number of elements in screen.
  • Each element of cols will be between 0 and M-1 inclusive, where M is the number of characters in each element of screen.
